# 玄言 / XuanLang

`玄言 / XuanLang` 是一门面向 AI 原生执行的中英融合高密度编程语言原型。它现在已经不是概念文档，而是一套可以实际运行、检查、格式化、编译、反汇编和测试的早期语言内核。

开发者署名：`开发者星尘_尘夜`

## 项目定位

玄言想做的不是“又一门普通脚本语言”，而是把下面几件事合在一起：

- 用中文语义作为主要表达层，降低中文场景下的语义割裂。
- 用编译器前端、IR、三条执行后端把语言链路先真正跑通。
- 用张量、KV Cache、attention、transformer step 等运行时原型，把语言和 AI 推理场景提前接上。
- 用结构体、映射、成员赋值、索引赋值这些基础能力，把它变成可继续演进的工程底座。

如果用一句话概括当前版本：

> 玄言已经是一门“可执行、可扩展、可继续进化”的 AI 原生语言原型，而不是停留在设想里的语法草稿。

## 当前状态

当前仓库可以直接作为开源首发版本发布，原因很简单：它已经具备一整条完整可验证的链路。

- 完整前端：词法分析、语法分析、AST、语义分析、基础优化、IR。
- 三条执行链：解释器执行、Python 后端执行、独立 VM 字节码执行。
- 核心语言能力：变量、常量、函数、条件、循环、模块导入、匹配、静态类型标注与类型推导。
- 复合数据能力：列表、映射、成员访问、索引访问、复杂左值赋值、结构体声明与实例化。
- AI 运行时原型：张量、矩阵乘法、量化入口、KV Cache、attention、transformer step。
- 工具链：`run`、`runpy`、`runvm`、`build`、`fmt`、`tokens`、`ast`、`optast`、`ir`、`asm`、`symbols`、`py2xuan`、`new`、`test`。
- 工程配套：示例、项目脚手架、自动化测试、架构文档、路线图、贡献指南、安全说明。

截至当前版本，仓库内已经附带 `30` 项自动化测试，用于验证解释器、Python 后端、VM 后端、项目工具和核心语法能力。

## 当前已经能做什么

### 1. 写正常的语言程序

- 定义变量和常量
- 写函数和返回值
- 使用条件、循环、匹配分支
- 导入模块并组织项目
- 用类型标注约束程序结构

### 2. 操作复合数据

- 列表与映射字面量
- 索引读取与赋值
- 成员读取与赋值
- 结构体类型定义与结构实例化
- 结构体和映射混合嵌套使用

### 3. 做编译链相关工作

- 看词法 Token
- 看 AST 和优化后的 AST
- 看简化 IR
- 看 VM 字节码反汇编
- 把玄言源码编译成可执行的 Python 代码

### 4. 做 AI 方向的原型实验

- 试验张量表达
- 做矩阵乘法和基础数值操作
- 使用 KV Cache 与 attention 原型
- 在语言层验证后续推理运行时的语义设计

## 一个实际示例

```xuan
构 引擎 {
    名称: 文
    版本: 整
    配置: 映[文, 文]
}

术 升级(目标: 引擎) -> 整 {
    目标.版本 = 目标.版本 + 1；
    目标.配置["后端"] = "vm"；
    返 目标.版本；
}

令 核心: 引擎 = 新 引擎 {
    名称: "玄言核",
    版本: 1,
    配置: {"后端": "解释器"}
}；

显("升级前:", 核心)；
令 版本 = 升级(核心)；
显("升级后:", 核心, 版本)；
```

这个例子背后实际覆盖了：结构体、映射、函数、类型标注、成员赋值、索引赋值和标准输出。

## 核心特性

| 模块 | 当前状态 | 说明 |
| --- | --- | --- |
| 语言前端 | 已实现 | Lexer、Parser、AST、Semantic、Optimizer、IR |
| 解释执行 | 已实现 | 可直接执行玄言源码 |
| Python 后端 | 已实现 | 可编译成 Python 并执行 |
| VM 后端 | 已实现 | 有独立字节码和反汇编能力 |
| 类型系统 | 初步可用 | 静态标注、推导、映射与结构体类型 |
| 数据结构 | 初步可用 | 列表、映射、结构体 |
| AI 运行时 | 原型可用 | 张量、量化入口、KV Cache、attention、transformer step |
| 工程化工具 | 已实现 | 新建项目、运行、检查、格式化、测试 |
| 生态系统 | 早期阶段 | 还没有包管理器、调试器、编辑器插件 |
| 原生硬件后端 | 未实现 | AOT/JIT、CUDA/ROCm/NPU 仍在路线图里 |

## 为什么说它现在已经值得开源

因为它已经具备了一个开源语言项目最关键的几个条件：

- 不是只有语法截图，而是能跑真实源码。
- 不是只有单一解释器，而是有三条执行链，方便后续演进。
- 不是只有语言壳子，而是已经有 AI 方向运行时原型。
- 不是只有源码堆积，而是带测试、文档、路线图和脚手架。
- 不是把未来说得很满，而是把已经实现的部分落成了可以验证的工程。

## 当前边界

这仍然是“早期但可运行的语言内核”，还不是终局版本。当前明确还没有完成：

- 真正的 AOT / JIT 双引擎
- 原生机器码后端
- 全平台硬件后端适配
- 完整的 C / C++ / Python 双向 ABI
- 正式包管理器、调试器、编辑器插件生态
- 面向生产的大规模标准库体系

所以它现在更适合：

- 语言设计与编译器实验
- AI 运行时语义验证
- 中文编程表达探索
- VM / 后端 / IR 演进实验
- 作为未来更大目标的底座继续迭代
