# 玄言全栈蓝图

这份蓝图把终极目标拆成可连续实现的工程层次，避免语言项目沦为“只有语法，没有系统”的空架子。

## 1. 语言核心

- 中英融合语法
- 静态类型 + 推导
- 函数、模块、控制流、张量表达式
- 编译期执行与元编程预留点

当前状态：

- 已有词法、语法、AST、基础语义
- 已支持变量、常量、函数、条件、循环、导入、张量操作

## 2. 编译器

- Lexer
- Parser
- Semantic
- AST
- IR
- 优化器
- 后端生成器

当前状态：

- 前五层已成型
- 当前后端为 Python 代码生成器
- AOT/JIT 仍待实现

## 3. Runtime

- 模块加载器
- Builtins
- 张量执行
- 量化入口
- 缓存与热更新
- 系统调用与硬件调度

当前状态：

- 预载运行时、模块加载、张量和量化入口已可用
- 系统调用封装、高性能缓存、热更调度未完成

## 4. 标准库

- 数学
- 张量
- 系统
- IO / 网络 / 加密 / 压缩

当前状态：

- 数学、张量、系统已建入口
- IO/网络/加密/压缩待扩展

## 5. AI 原生核心

- 张量类型
- MatMul / Softmax / ReLU
- KV Cache
- Attention
- Transformer / Mamba 算子
- 量化与稀疏化

当前状态：

- 已有基础张量、MatMul、Softmax、INT4 量化入口
- Attention / KV Cache / 模型格式加载待实现

## 6. 硬件与平台

- Windows / Linux / Android / macOS
- x86_64 / ARM64
- GPU / NPU 后端
- Root / 管理员资源控制

当前状态：

- 当前版本为平台无关的 Python 原型
- 后续将按 Android / ARM64 / 手机 AI 场景优先推进

## 7. 外部转译与兼容

- Python -> 玄言
- C/C++ -> 玄言
- 自然语言 -> 玄言
- GGUF / ONNX / Safetensors 接入

当前状态：

- Python 子集转译器已落地
- 其余转译器仍待实现

## 8. 类量子扩展

- 叠加态数据类型
- 路径采样语义
- 概率态优化
- 非二进制高密度执行模型

当前状态：

- 当前只保留设计位
- 会放在 IR 与 Runtime 的下一轮升级里
