# 玄言架构

当前版本采用可快速迭代的五层结构：

1. `lexer`
   - 负责中英混排源码的标准化与切词
   - 同时支持半角/全角标点
2. `parser`
   - 递归下降语法分析
   - 生成强类型 AST
3. `semantic`
   - 作用域、符号表、基础类型检查
   - 对变量、常量、函数返回、赋值进行一致性验证
4. `ir`
   - 降级为可观察的线性中间表示
   - 为后续优化器和 AOT/JIT 后端保留位置
5. `runtime + codegen`
   - 当前先落到 Python 代码生成后端
   - 运行时负责张量、量化、标准库、动态导入

这条路线的意义是：现在先把语言“活过来”，后面再逐步替换成更底层、更快的执行后端。
