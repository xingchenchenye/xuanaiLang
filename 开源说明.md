# 玄言开源说明

这份文档的目标很直接：让第一次看到 `玄言 / XuanLang` 的人，能在几分钟内知道它是什么、已经做到什么程度、强在哪里、边界又在哪里。

## 一句话介绍

玄言是一门面向 AI 原生执行场景设计的中英融合高密度编程语言原型，当前已经具备完整前端、解释器/Python/VM 三后端、结构化数据能力和 AI 运行时原型。

## 这个项目的作用

玄言目前主要承担四个作用：

1. 作为一门真实可运行的语言内核，验证中文语义与高密度表达的设计方向。
2. 作为一套编译器工程底座，持续扩展类型系统、IR、执行后端和工具链。
3. 作为 AI 运行时实验场，把张量、KV Cache、attention、transformer step 等能力前置到语言层。
4. 作为未来更强执行系统的起点，为 AOT、JIT、ARM、Android、GPU/NPU 适配预留接口。

## 这门语言现在强在哪里

这里说“强”，只说现在仓库里已经真实落下来的部分。

### 1. 它不是语法演示，而是完整链路

- 有 Lexer、Parser、AST、Semantic、Optimizer、IR。
- 有解释器执行链。
- 有 Python 编译后端。
- 有独立 VM 字节码执行后端和反汇编能力。

很多语言原型只停在“能解析一段语法”，玄言已经走到“能从源码一路走到多后端执行”的阶段。

### 2. 它已经有比较完整的核心语言形态

- 变量、常量、函数、返回值
- 条件、循环、匹配
- 类型标注与类型推导
- 模块导入
- 列表、映射、结构体
- 成员访问、索引访问、复杂左值赋值

这意味着它已经可以承载比“玩具语言”更像样的程序结构。

### 3. 它提前把 AI 运行时接进来了

- 张量
- 矩阵乘法
- 量化入口
- KV Cache
- attention
- transformer step

这部分还属于原型层，但方向很明确：玄言不是单纯给传统业务逻辑写脚本，它从一开始就朝着 AI 计算与推理表达去设计。

### 4. 它已经具备开源协作最基本的工程条件

- README
- 路线图
- 架构文档
- 示例
- 自动化测试
- 项目脚手架
- 贡献与安全说明

也就是说，现在开源出去，别人是能读、能跑、能理解、能贡献的。

## 当前适合拿它做什么

- 研究中文语义编程语言设计
- 做编译器和虚拟机方向的实验
- 验证 AI 运行时表达模型
- 作为未来自定义执行系统的前期底座
- 作为个人或小团队的长期语言工程项目起点

## 当前还不适合宣称什么

为了保证对外说法足够真实，下面这些能力现在还不能说已经完成：

- 生产级通用语言生态
- 真正的 AOT / JIT 双引擎
- 原生机器码生成
- CUDA / ROCm / NPU 等硬件后端
- 完整跨平台运行时适配
- 成熟的标准包管理和第三方生态

所以更准确的表述应该是：

> 玄言已经是一门可运行、可扩展、可验证的早期语言内核，而不是面向生产的成熟通用语言。

## 当前版本最值得展示的点

如果你要给别人演示，优先展示这几样：

1. `examples/快速开始.xy`
2. `examples/映射演示.xy`
3. `examples/结构演示.xy`
4. `py -3 -m xuanlang asm .\examples\快速开始.xy`
5. `py -3 -m unittest discover -s tests -v`

这几项能最直观地说明：玄言不是嘴上说说，而是真的已经能跑起来。
